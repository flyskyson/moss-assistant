# MOSS Assistant 开发工作流

## 日常开发流程

```bash
# 1. 启动 MOSS
python start_moss.py

# 2. 使用和测试

# 3. 如果有好的想法或发现问题，记录下来

# 4. 告诉 Claude，一起实现

# 5. 测试新功能

# 6. 提交代码
git add .
git commit -m "描述你的改动"
git push
```

## 和 Claude 协作的模式

### 模式 1: 优化现有功能
```
你: "我觉得导师角色太严厉了"
Claude: "我来帮你调整 prompt..."
你: "测试一下效果..."
Claude: "根据反馈继续优化..."
```

### 模式 2: 添加新功能
```
你: "我想添加一个功能..."
Claude: "好的，让我们规划一下..."
     1. 设计功能
     2. 编写代码
     3. 测试验证
     4. 部署上线
```

### 模式 3: 修复问题
```
你: "出现了一个错误..."
Claude: "让我分析一下..."
     "找到问题了，修复方案是..."
你: "修复成功！"
```

## 优先级管理

### 高优先级（立即处理）
- 💥 系统崩溃
- 🔐 数据丢失
- 🚫 安全问题

### 中优先级（本周处理）
- ✨ 功能优化
- 🐛 Bug 修复
- 📝 文档更新

### 低优先级（有空再做）
- 🎨 界面美化
- 🚀 新功能探索
- 📊 性能优化

## 持续改进建议

1. **每次对话后记录想法**
   - 哪些回应很好？
   - 哪些需要改进？
   - 想要什么新功能？

2. **定期和 Claude 讨论**
   - 分享使用体验
   - 提出改进建议
   - 探索新可能性

3. **实验新功能**
   - 在分支上测试
   - 验证效果
   - 合并或放弃

4. **保持代码整洁**
   - 及时提交改动
   - 写好 commit 说明
   - 更新文档
