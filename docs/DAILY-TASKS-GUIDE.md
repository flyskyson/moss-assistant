## 🔄 两种任务类型

### 1️⃣ 每日例行任务

**特点**: 每天都要做的事情

**示例**:
- `newday` - 创建今日记忆文件
- `check-system-status.sh` - 检查系统状态
- 备份数据
- 查看邮件

**提醒方式**: MOSS 每次初始化时都会提醒

---

### 2️⃣ 一次性任务

**特点**: 完成后就可以标记为"已完成"

**示例**:
- 测试 MOSS 初始化（一次性）
- 知识库整理（本周完成）
- 创建 PARA 维护脚本（未来计划）

**提醒方式**: 完成后从列表中移除

## 💡 使用方法

### 添加每日例行任务

```
"添加每日例行任务：备份数据库，命令：~/backup.sh，时间：每天晚上"
```

**MOSS 会**:
1. 更新 TASKS.md
2. 添加到"每日例行任务"章节
3. 下次初始化时会提醒

---

### 执行每日例行任务

**方法 1: 让 MOSS 执行**

```
"帮我执行每日例行任务：newday"
```

**方法 2: 自己执行**

```bash
newday
```

---

### 标记一次性任务完成

```
"任务 '测试 MOSS 初始化' 完成了，从 TASKS.md 移除"
```

**MOSS 会**:
1. 从 TASKS.md 删除该任务
2. 移动到"已完成"章节
3. 下次不再提醒

---

### 查看所有任务

```
"今天有什么任务？"
```

**MOSS 会**:
1. 读取 TASKS.md
2. 列出所有未完成的任务
3. 按优先级排序

## 🔄 任务生命周期

### 每日例行任务

```
添加 → 每天提醒 → 执行 → 继续提醒（第二天）
```

**永远不会移除**，除非你手动删除

---

### 一次性任务

```
添加 → 提醒 → 执行 → 标记完成 → 移动到"已完成"
```

**完成后不再提醒**

---

## 💡 最佳实践

### 1. 每日例行任务保持简单

**好的每日例行任务**:
- ✅ newday（1 秒完成）
- ✅ 系统检查（10 秒完成）
- ✅ 查看今日计划（自动提醒）

**不好的每日例行任务**:
- ❌ 整理知识库（需要 30 分钟）
- ❌ 学习新技能（需要 1 小时）

**原因**: 每日例行任务应该是**快速、简单、必须**的

---

### 2. 一次性任务要明确截止时间

**好的描述**:
```
- 测试 MOSS 初始化（今天）
- 知识库整理（本周）
- 学习 Docker（本月）
```

**不好的描述**:
```
- 测试 MOSS 初始化
- 知识库整理
- 学习 Docker
```

---

### 3. 定期清理已完成任务

**每周日**:
1. 检查 TASKS.md
2. 清理"已完成"章节
3. 添加新任务
