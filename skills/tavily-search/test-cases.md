# MOSS 搜索功能测试用例

## 测试目标
验证 MOSS 是否能正确识别需要搜索的场景，并使用 Tavily API 获取准确信息。

## 测试用例

### 类别 1: 实时信息（必须搜索）
- [ ] "今天北京天气怎么样？"
- [ ] "2026年春节是哪天？"
- [ ] "最新的 AI 新闻有什么？"
- [ ] "苹果股票现在价格是多少？"

**预期**: MOSS 应该主动调用搜索，而不是只依赖训练数据

### 类别 2: 技术文档和 API
- [ ] "DeepSeek R1 模型的 API 怎么调用？"
- [ ] "OpenClaw 最新版本有什么新功能？"
- [ ] "Python 3.13 有什么新特性？"

**预期**: 搜索最新文档和更新日志

### 类别 3: 事实核查
- [ ] "地球是平的吗？"（应该能纠正）
- [ ] "2026年美国总统是谁？"（如果训练数据中没有）
- [ ] "马斯克收购推特了吗？"

**预期**: 搜索权威来源并给出准确答案

### 类别 4: 不需要搜索（应该使用本地知识）
- [ ] "Python 怎么定义函数？"
- [ ] "Git 如何创建分支？"
- [ ] "我的工作区有哪些文件？"

**预期**: 不应该调用搜索，直接回答或使用本地工具

### 类别 5: 多轮对话
- [ ] "搜索 DeepSeek 最新模型"
- [ ] "它有什么特点？"（追问，基于搜索结果）
- [ ] "和 GPT-4 相比如何？"（需要再次搜索对比）

**预期**: 能基于之前的搜索结果继续对话

### 类别 6: 中文和英文混合
- [ ] "What's the latest news about 特朗普？"
- [ ] "帮我搜索 Claude 4 Sonnet 的 features"

**预期**: 正确处理混合语言查询

## 评估标准

### ✅ 优秀
- 正确识别何时需要搜索
- 搜索结果准确且相关
- 能总结多个搜索结果
- 引用来源（URL）

### ⚠️ 需要改进
- 有搜索但结果不相关
- 没有总结，直接甩链接
- 不知道何时该搜索

### ❌ 失败
- 明明需要最新信息却不搜索
- 搜索脚本调用失败
- API 错误没有处理
